# Contributing to AWS CloudOps Lab

Thank you for contributing! This guide covers how students submit solutions and how instructors can improve the lab.

## For Students — Submitting a Solution

See [README.md](README.md#-how-to-submit) for the full submission guide.

**Quick rules:**
- One PR per project
- Branch name: `solution/project-N-your-name`
- Solution folder: `solutions/0N-project-name/your-github-username/`
- Must include `README.md` with screenshots
- Never commit AWS credentials or `.env` files

## For Instructors — Adding / Improving Projects

1. Fork and branch: `feat/project-13-new-topic`
2. Add under `projects/13-topic-name/README.md`
3. Include: objectives, architecture diagram (ASCII or image), required files list, rubric, cost estimate, cleanup steps, references
4. Update `index.html` `PROJECTS` array
5. Open a PR for review

## Code of Conduct

- Be respectful in PR reviews
- Give constructive feedback
- Help others in GitHub Discussions

## Security

If you find a security issue with the lab setup (not an AWS vulnerability), open a GitHub Issue marked `[SECURITY]`.
